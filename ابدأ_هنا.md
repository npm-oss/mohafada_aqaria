# 🚀 دليل البدء السريع - المحافظة العقارية

## المشكلة التي تم حلها
✅ تم حل مشكلة عدم الاتصال بقاعدة البيانات للأدمن والزبون

## السبب
المشروع يستخدم **Firebase Realtime Database** (وليس MySQL)، والمتصفحات تمنع تحميل JavaScript modules من الملفات المحلية مباشرة بسبب سياسة CORS.

## الحل (خطوة واحدة فقط!)

### على Windows:
1. افتح مجلد المشروع
2. انتقل إلى: `public/firebase-app`
3. انقر مرتين على ملف: **`start-server.bat`**
4. سيفتح نافذة سوداء (لا تغلقها!)

### على Linux/Mac:
```bash
cd public/firebase-app
chmod +x start-server.sh
./start-server.sh
```

## 🌐 افتح المتصفح

بعد تشغيل السيرفر، افتح:

### للزبون (العملاء):
```
http://localhost:8000/index.html
```

### للأدمن (الإدارة):
```
http://localhost:8000/admin/login.html
```

### لاختبار الاتصال:
```
http://localhost:8000/test-connection.html
```

## 🔐 تسجيل الدخول للأدمن

يجب إنشاء حساب أدمن أولاً:

1. اذهب إلى: https://console.firebase.google.com
2. اختر المشروع: `project-811172743097267139`
3. من القائمة الجانبية: **Authentication**
4. اضغط **Add User**
5. أدخل:
   - البريد الإلكتروني: `admin@mohafada.dz` (مثال)
   - كلمة المرور: `Admin@123` (مثال)
6. احفظ البيانات

الآن يمكنك تسجيل الدخول بهذه البيانات!

## ⚠️ مهم جداً

### ✅ افعل:
- شغّل السيرفر المحلي دائماً
- استخدم `http://localhost:8000`
- تأكد من اتصال الإنترنت

### ❌ لا تفعل:
- لا تفتح الملفات مباشرة من المجلد
- لا تستخدم `file:///` في المتصفح
- لا تغلق نافذة السيرفر أثناء العمل

## 🔧 إذا لم يعمل

### 1. تحقق من السيرفر
تأكد أن السيرفر يعمل (النافذة السوداء مفتوحة)

### 2. اختبر الاتصال
افتح: http://localhost:8000/test-connection.html

يجب أن ترى 5 علامات ✅ خضراء

### 3. تحقق من Firebase Console
اذهب إلى: https://console.firebase.google.com
- تأكد أن المشروع نشط
- تحقق من قواعد Database:

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

### 4. تحقق من Console المتصفح
اضغط `F12` في المتصفح وشاهد الأخطاء
